# Product Requirements Document (PRD): v5.0 LiteLLM & Whisper Integration

## 1. Overview
The goal of this revision is to transition CubeBot from direct API connections to a centralized, self-hosted, intelligent AI gateway using **LiteLLM**. This architecture will be hosted on the Nvidia Jetson Orin Nano via Tailscale. Further, we are migrating voice transcription from browser-dependent APIs to a fully open-source **Whisper server** running locally on the Jetson.

## 2. Objectives & Features
### 2.1 Unified LiteLLM Gateway
- **Intelligent Routing**: Route to Kimi API, Groq API, and local Ollama from a single gateway.
- **RAG & Fallback Strategy**: 
  - Primary Router: Attempt Groq (Ultra-Fast) for reasoning tasks.
  - Fallback 1: If rate-limited, fall back to Kimi (Moonshot).
  - Fallback 2: Local Ollama (e.g., Llama 3) if offline entirely.
- **Multi-Tenant Accounts**: 
  - Ability to generate API keys for friends/users without exposing core provider keys.
  - Track cost and tokens independently using the LiteLLM proxy dashboard.
- **Docker Setup**: Add `litellm-proxy` to a Docker Compose configuration.

### 2.2 Voice & Transcription Overhaul
- **Open-Source Whisper Migration**: Replace browser Web Speech API `SpeechRecognition` with a local Whisper instance (e.g., `whisper.cpp` or `faster-whisper` Docker container).
- **Audio Recording**: The web app will capture microphone blobs via MediaRecorder, send them over HTTP to the Jetson API, and receive transcribed text back.
- **Fix 30-sec Voice TTS Hang Strategy**: 
  - The current Chrome TTS `pause/resume` hack periodically fails, leaving the synthesis engine in a zombie "Transmitting" state.
  - **Proposed Fix**: Instead of passing one giant 1000-character paragraph to `SpeechSynthesisUtterance`, we will split the AI response by sentence boundaries (e.g. `.` `?` `!`). We will enqueue these smaller sentences as consecutive, short utterances. The browser completes short utterances reliably.

### 2.3 Settings UI Enhancements
- **LiteLLM Dashboard Link**: Provide a quick link out to the observability dashboard running on Jetson `port 4000/ui`.
- **Custom Router Strategy Selection**: Expose toggles/dropdowns for "Fastest Response (Groq)", "Deep Context (Kimi)", or "Local Only (Ollama)".
- **Whisper URL Field**: Input field to point STT calls to the local Whisper API.

## 3. Architecture Changes (Jetson Side)
The Jetson Orin Nano (Tailscale IP currently `100.83.247.26`) will host:
1.  **Ollama**: `port 11434`
2.  **LiteLLM Proxy**: `port 4000`
3.  **Whisper Server**: `port 9000` (FastAPI or similar wrapper)

## 4. Documentation Standards
- Each major revision (v4.0, v5.0) gets its own documentation file appended in `/docs/revisions/`.
- The root `README.md` acts as a historical log and directory mapping exactly how to rebuild the system from scratch, ensuring open-source maintainability.
